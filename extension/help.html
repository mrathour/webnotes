WebNotes

Capture any webpage element → AI notes → Notion

How it works

Pick Element
›
Saved as .txt
›
AI generates .md note
›
Synced to Notion

Click any element on a webpage to save its content locally. The Sync Notes with Notion button runs AI to turn raw captures into structured revision-ready notes, then pushes each note as a new page into your chosen Notion database or page.

Setup

1
Start the backend server

Open a terminal in the backend/ folder and run:

uv run uvicorn main:app --reload

The server must be running at http://localhost:8000 for the extension to work.

2
Connect to Notion

Click Connect to Notion in the popup and follow the OAuth flow. You'll be asked to grant access to your workspace.

3
Select a target

After connecting, choose a Notion page or database from the dropdown. Notes will be created there.

  • [DB] — rows appear as entries in a database
  • [Page] — notes appear as sub-pages

Capturing elements

1
Activate the picker

Click Pick Element. A crosshair cursor appears and elements highlight as you hover.

2
Click to capture

Click any highlighted element. Its text content and HTML are saved to backend/saved/ as a .txt file named after the page title.

Capturing the same page title again overwrites the previous file — useful for refreshing content.
3
Cancel anytime

Press Esc to exit pick mode without capturing.

Syncing notes to Notion

1
Click "Sync Notes with Notion"

The button finds all captures that don't have a note yet, runs AI on each one to produce a structured .md file, then pushes every new note to Notion as a page.

2
Watch the live log

A real-time log appears below the button as each file is processed. Green text = success. Red text = error with a reason.

3
Notes land in Notion

Each note becomes a new page with headings, code blocks, tables, and checklists preserved. Already-synced captures are skipped automatically.

AI note generation can take 30–60 seconds per capture. Leave the popup open until the log shows completion.

Troubleshooting

Backend is not running error

Start the server: open a terminal in backend/ and run uv run uvicorn main:app --reload. Make sure uv is installed and dependencies are synced (uv sync).

Connect to Notion button does nothing or fails

Ensure NOTION_CLIENT_ID and NOTION_CLIENT_SECRET are set in backend/.env and the server is running. The backend handles the OAuth token exchange.

No pages or databases appear in the dropdown

Your Notion integration must be explicitly connected to a page or database in Notion. Open Notion, go to the page/database, click ··· → Connections, and add your integration.

Sync finishes but nothing appears in Notion

Check that the selected target (shown in the popup) still exists in Notion and the integration has access. Try clicking change and re-selecting the target.

Toast notification does not appear after capturing

The backend is likely not running. Start the server and try again.
